# iPhoneで遊べるようにする手順（公開ガイド）

このゲームは **PWA（ウェブアプリ）** です。App Store は不要で、
**「ファイルを無料でネットに公開（HTTPS）」→「iPhoneのSafariで開く」→「ホーム画面に追加」**
の流れで、アプリのように全画面で遊べます。

公開するフォルダ: **`C:\Users\kunit\kids-word-game`**
（ドラッグ用のzipも用意: `C:\Users\kunit\kids-word-game-deploy.zip`）

---

## ⭐ いちばん簡単な方法：Netlify Drop（アカウント不要で即公開）

パソコンのブラウザだけで、ドラッグ＆ドロップで完了します。

1. パソコンのブラウザで **https://app.netlify.com/drop** を開く（ログイン不要）
2. エクスプローラーで **`C:\Users\kunit\kids-word-game`** フォルダを開く
3. その**フォルダごと**（または中のファイル全部）を、Netlifyの点線の枠に**ドラッグ＆ドロップ**
4. 10〜30秒待つと、`https://〇〇〇〇.netlify.app` という**公開URL**が表示される ← これがゲームのアドレス
5. **【重要】そのページの「Sign up」/「Claim this site」からメールで無料登録**して、このサイトを"自分のもの"にする
   - ※登録しないと **24時間で自動的に消えます**。無料登録すればずっと残ります（クレカ不要）。
6. 表示されたURLを、iPhoneのSafariで開く（次の「iPhoneで遊ぶ」へ）

**あとで中身を直したいとき**：このゲームのファイルを編集 → もう一度 https://app.netlify.com/drop にフォルダをドラッグ（登録済みなら管理画面からも差し替え可）。

## 📱 iPhoneで遊ぶ（ホーム画面に追加）

1. iPhoneの **Safari** で、上で公開したURLを開く
2. 画面下の **共有ボタン**（□に↑のマーク）をタップ
3. メニューを下にスクロールして **「ホーム画面に追加」** をタップ
4. 名前（例：「ことばあそび」）を確認して、右上の **「追加」** をタップ
5. ホーム画面にできたアイコンから起動すると、**全画面**で遊べます
6. 最初に一度画面をタッチすると、そこから音・声が鳴り始めます（iPhoneの仕様）

### ⚠️ 音についての注意（iOS 26 系の既知の不具合）
- iPhoneの最近のOS（iOS 26 系）では、**ホーム画面アプリ（全画面）として長時間使うと、音が止まることがある**不具合が報告されています（2026年5月時点で未修正）。
- もし**音が出なくなったら**：
  - アプリをいったん閉じてから開き直す（多くの場合これで復活します。自動復帰の保険も入れてあります）
  - それでもダメなら、**Safariのタブで同じURLを開いて遊ぶ**と音が安定します
    （Safariのタブに「ブックマーク」しておくと、すぐ開けて便利です）
- 「音の確実さ」を最優先するなら、**Safariのタブ（ブックマーク）で遊ぶ**のがいちばん安定します。
  全画面の見た目を優先するなら「ホーム画面に追加」。お好みでどうぞ。

## 🔁 もう一つの方法：GitHub Pages（無料・ずっと残る・更新も簡単）

「ちゃんと保存して長く使いたい」「あとで何度も更新したい」なら GitHub Pages もおすすめ。
（GitHubアカウントの作成が必要。ターミナル/gitは不要、ブラウザだけでOK）

1. **github.com** でアカウント作成（またはログイン）
2. 右上の「＋」→「New repository」。名前を `kids-word-game`、公開（Public）を選んで「Create repository」
3. リポジトリ画面で「**Add file → Upload files**」をクリックし、
   `index.html` / `manifest.json` / `icon-180.png` / `icon-192.png` / `icon-512.png` / `icon.svg`
   をまとめてドラッグ → 下の「Commit changes」
4. 上部の「**Settings**」→ 左メニュー「**Pages**」→ 「Branch」を `main` ・フォルダ `/ (root)` にして「Save」
5. 数分〜最大20分ほどで、`https://（あなたのユーザー名）.github.io/kids-word-game` が公開される
6. そのURLをiPhoneのSafariで開いて、上の「ホーム画面に追加」へ

**あとで更新**：同じリポジトリで「Add file → Upload files」で新しいファイルを上げるだけ。数分で反映されます。

## どっちを選ぶ？

| | Netlify Drop | GitHub Pages |
|---|---|---|
| 手軽さ | ◎ ドラッグだけ・登録は後でOK | ○ アカウント作成が必要 |
| 公開の速さ | ◎ 数十秒 | ○ 数分〜20分 |
| ずっと残す | 無料登録すればOK | ◎ 標準で永続 |
| 更新のしやすさ | ○ 再ドラッグ | ◎ ファイル差し替え |
| URLの例 | `xxxx.netlify.app` | `ユーザー名.github.io/kids-word-game` |

**まずはNetlify Dropで試して、気に入ったらGitHub Pagesに引っ越す**のがおすすめです。

## 将来も使うなら：サービスの選び方（汎用性 × 安全性）2026年版

「この2つ（Netlify / GitHub Pages）だけ？」 → **いいえ。** 主要なものだけで10前後あります。
「今後も汎用的に使えて・安全」を基準にすると、おすすめは次のとおり。

| サービス | 提供元 | 汎用性（将来フルアプリ化） | 安全性・継続性 | 無料/商用 | 簡単さ |
|---|---|---|---|---|---|
| **Cloudflare Pages** ⭐総合1位 | Cloudflare（上場） | ◎ Workers/DB/ストレージまで拡張可 | ◎ DDoS/WAF標準・上場企業 | 帯域無制限・商用OK | ○ |
| **Firebase Hosting** | Google | ◎ ログイン/DB/サーバー機能が一体 | ◎ Google級 | 無料枠あり・商用OK | △〜○ |
| **GitHub Pages** | Microsoft | △ 静的のみ（裏側機能なし） | ◎ 必須2段階認証・無事故 | 無料・基本OK | ◎一番簡単 |
| **Netlify** | Netlify社（非上場） | ○ 関数等あり | ○ 事故なし・会社は中規模 | 商用OK・2025年からクレジット課金 | ◎ |
| Vercel | Vercel社（非上場） | ○ Next.js向き | △ 2026/4にサプライチェーン侵害 | **無料は商用NG**（$20/月〜） | ○ |

### 結論（おすすめ）
- **将来いろいろ作るかも＋安全重視で1つに絞るなら → Cloudflare Pages**
  （無料で帯域無制限・商用OK、DDoS/WAF標準装備、後からログインやDBも足せる、上場企業で継続性◎）
- **アプリにログインやデータ保存を付けたくなりそう → Firebase（Google）**（一体型で強力。少し学習が必要）
- **とにかく簡単＆大手の安心で、静的のままでいい → GitHub Pages（Microsoft）**

### 注意
- **Vercel**：無料プランは「商用利用NG」。将来お金が絡むなら無料では選ばない方が無難。2026年4月に外部経由の侵害あり。
- **Netlify**：始めるのは一番ラク・商用もOKだが、会社規模が小さめでクレジット課金により費用が読みにくい面あり（少量利用なら問題なし）。
- どのサービスでも、**自分のアカウントに2段階認証（パスキー）を設定**しておくと安全性が大きく上がります。

> ※「このゲームを置くだけ」ならどれも十分安全です。違いが効いてくるのは“将来アプリに育てる/長く一本化する”とき。

## 困ったときのチェック
- 公開URLが **`https://`** で始まっているか（PWAにはHTTPSが必須。上記サービスは自動でHTTPSになります）
- iPhoneは **Safari** で開く（Chrome等だと「ホーム画面に追加」が出ない場合あり）
- 音が出ない → 一度画面をタッチ／アプリを開き直す／Safariのタブで開く
